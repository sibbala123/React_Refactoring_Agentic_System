Empirical Methodology
Experiment 1
Do React Refactors Preserve UI Behaviour?
1
Synthetic React App
Custom project built with ~20 deliberate React code smells across 3 routes and 8 distinct UI states
2
Capture Baseline
Before refactoring, capture screenshots, full DOM snapshots, and truth documents recording the position and presence of every UI component per state
3
Apply Refactors
Strong LLM refactors each smell using carefully constructed prompts with full file context. All code smells are resolved by the pipeline
4
Capture Post-Refactor
Same 8 states re-captured: screenshots, DOM snapshots, and truth documents after all refactors are applied
5
Compare
Diff screenshots pixel-by-pixel, compare DOM structure, and validate truth document component positions before vs after
3 RoutesHome, Customers, Reports
8 UI Statesdefault, filtered, selected, help open, opened report, pinned ready
3 Artefact TypesPNG screenshots, HTML DOM snapshots, JSON truth documents
GoalConfirm structural refactors produce zero observable UI change
Experiment 2
Agentic Pipeline vs. Weak LLM vs. Strong LLM (20 Smells)
Corpus
20 smells, stratified random sample from 1,210 actionable smells in the Supabase codebase (seed = 42)
LLM Config
temperature=0 across all systems for reproducibility. Pipeline uses GPT-4o-mini for classify, plan, critique and GPT-4o for edit
Pipeline Gates
tsc --noEmit, ReactSniffer re-run, pnpm build, critique score above 0.7, up to 3 retries
System Model Approach Evaluation Method
Weak LLM GPT-3.5-turbo Direct neutral prompt. No tactic guidance, no verification step, single shot LLM-as-judge
Strong LLM GPT-4o Direct neutral prompt. No tactic guidance, no verification step, single shot LLM-as-judge
Agentic Pipeline GPT-4o-mini + GPT-4o Classify, Plan (tactic-constrained), Edit (bounded scope), Verify, Critique with retry loop Automated gates
Experiment 3
End-to-End VS Code Extension Runs on Real-World Repositories
ToolReactRefactor VS Code extension, end-to-end from scan to accepted fix
Repo SelectionThree open-source React projects chosen to represent small, medium, and large codebase scales
ProcedureScan each repo, select smells using the extension, run the full agentic pipeline, record accepted, rejected, skipped, and failed outcomes
Small
JUMBLE
github.com/Shikha307/JUMBLE
23 identified code smells
Small hobby-scale React project. Used to validate the pipeline on a minimal codebase with limited component complexity.
Medium
Excalidraw
github.com/excalidraw/excalidraw
101 identified code smells
Widely-used open-source diagramming tool. Represents a mid-scale production React codebase with diverse component patterns.
Large
Supabase
github.com/supabase/supabase
1,137 identified code smells
Large-scale production monorepo with modern React patterns including shadcn/ui, react-hook-form, and Radix UI primitives.

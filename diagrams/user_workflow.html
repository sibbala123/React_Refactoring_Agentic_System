ReactRefactor, User Interaction Workflow
From Installation to Refactoring
Phase 1 Setup and Configuration
Install the Extension
Install from VSIX in VS Code or from the Marketplace. Requires VS Code 1.85 or higher.
VSIX
Configure OpenAI API Key
Set via Settings. Stored in VS Code secret storage. Falls back to the OPENAI_API_KEY environment variable if left empty.
Optional
Open a React Workspace, Server Starts
Auto-activates on .tsx, .jsx, or .js files. Python 3.8 or higher is detected, the stale port 7432 process is killed, and the FastAPI server is spawned.
Port 7432
Phase 2 Scanning
Click Scan in the Sidebar
Scan button in the Code Smells toolbar. Active filter is reset and a loading indicator shows while the backend runs.
POST /scan
Server Detects Code Smells
ReactSniffer CLI runs AST analysis and outputs CSV files. Heuristic fallback activates if no CSV is produced. Build dirs and node_modules are excluded. Library paths like design-system are downgraded to low severity.
CLI
Smells Appear in Tree View
Grouped by file and sorted by severity. Each leaf shows smell type, component name, and line range with a severity icon.
Tree View
Phase 3 Selection and Filtering
Manual Checkbox
Click individual smells or a file checkbox to select all children at once.
Select Top 100
Selects the 100 highest-severity smells sorted high to low in one click.
Filter by Type
Multi-select dropdown to narrow the tree view by smell type.
Go to Smell
Click any unfixed smell to jump to its exact line in the editor.
Any smells selected?
Phase 4 Fix Pipeline
Click Fix Selected
Wrench button is enabled only when smells are selected and no fix is running. A cancel button replaces it while the pipeline is active.
POST /fix
Pipeline Runs Per Smell
Each smell passes through Classify, Plan, Edit, Verify, Critique, and Finalize nodes. If the critique score is too low, the smell loops back to Plan for another attempt.
SSE stream
Phase 5 Results and Post-Fix Actions
Accepted
Fix passed critique. Code changes were applied to the file on disk.
Rejected
Pipeline could not produce a passing fix after exhausting all retry attempts.
Skipped
Classified as low confidence or out of scope. Requires manual review.
Failed
A pipeline error occurred, such as an LLM timeout or a parse failure.
Post-Fix Actions
Click an accepted or rejected smell to open a VS Code diff editor. It compares the original file fetched via git show HEAD against the current file on disk. This requires the workspace to be a git-initialized repository.

Endpoint used: GET /original
Right-click an accepted smell and select Revert Fix to restore the original file content from the backup saved before the fix was applied. The smell status resets and it becomes selectable again.

Endpoint used: POST /revert
Hover over any smell to see a detailed tooltip showing the smell type, component name, line range, severity level, number of retry attempts, critique score, and the rejection reason if the fix was rejected.
Run another Scan to pick up newly introduced smells, or change the filter and fix a different smell type in a new batch.

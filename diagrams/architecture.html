Server Internals
⚙️
_apply_severity()
Post-processes smell list. Assigns severity by smell type but skips smells already marked low by the detector (library paths preserved).
📏
_enrich_smell_lines()
Parses source files to find exact start/end line numbers for named components when the detector omits line info.
🗂️
Job State Store
In-memory dict of active jobs. Each holds a task list, per-task status, backup file paths, and an async SSE event queue. Cleaned up on job completion.
subprocess · CSV output · smell dicts
Layer 3 — Smell Detection
🔬
detect_smells.py
agentic_refactor_system/scripts/
Orchestrates detection. Calls ReactSniffer CLI, reads components_smells.csv and files_smells.csv, deduplicates, applies path filters and severity overrides.
⚛️
ReactSniffer CLI
vendor/reactsniffer/
Node.js CLI bundled in the VSIX (whitelisted from node_modules exclusion in bundle.js and .vscodeignore). AST-based analysis, outputs two CSV files.
AST-based CSV output
🛡️
Heuristic Fallback
detect_smells.py
Activates when ReactSniffer produces no CSV output. File-based heuristics estimate smells. Less precise — noted in output channel logs.
🏷️
Path Classifier
detect_smells.py
Excluded entirely: node_modules/, out/, __generated__/, .cache/

Downgraded → low: packages/*/, design-system/, ui-kit/, ui-components/, component-library/, primitives/, base-components/
Generalizable Regex patterns

Supporting Modules
🗺️
state.py
LangGraph state schema. Carries smell metadata, plan, edited code, verify result, critique score, and retry count between nodes.
🔄
runner.py
Async runner that executes the graph per-smell. Emits progress events. Respects cancellation flag from /cancel endpoint.
💡
tactics.py
Smell-type → fix strategy lookup. Maps each smell type to a recommended refactoring tactic fed to the planner.
📦
ingestion/
Context-gathering utilities. Reads file contents, extracts component code, builds per-smell prompt payloads for the LLM nodes.
